# 웹밴드 스튜디오

브라우저에서 바로 쓰는 교육용 음악 제작 스튜디오입니다. 학생은 루프, 미디, 녹음 클립으로 곡을 만들고 과제로 제출할 수 있으며, 교사는 반, 학생, 과제, 레슨, 제출 현황을 관리할 수 있습니다.

## 실행

```bash
npm install
npm run dev
```

로컬 개발 서버는 `http://127.0.0.1:3000`에서 열립니다.

## 빌드

```bash
npm run build
npm run qa:smoke
```

빌드는 GitHub Pages의 상대 경로 배포를 위해 `base: ./`로 생성됩니다.

## 수업 흐름

1. 교사 보기에서 반을 만들고 학생을 등록합니다.
2. 기본 레슨 또는 레슨 빌더로 만든 커스텀 레슨을 과제에 연결합니다.
3. 학생 보기에서 학생을 선택하고 과제를 시작합니다.
4. 학생은 편집기에서 루프, 미디, 녹음, 샘플을 배치합니다.
5. 검토 모드에서 자동 점검, 루브릭 점수, 다음 행동 추천을 확인합니다.
6. 제출 패키지를 만들면 `.webband.json`, 리뷰 요약 JSON, WAV 파일이 내려받아지고 제출 이력이 저장됩니다.
7. 교사는 제출 상세에서 점수, 경고, 미션 완료율, 재제출 횟수, 피드백을 확인하고 CSV로 내보낼 수 있습니다.

## 저장 방식

- 기본값은 Dexie 기반 IndexedDB 로컬 저장입니다.
- 모의 클라우드 모드는 같은 repository 인터페이스를 `localStorage`와 IndexedDB로 흉내 내어 수업 흐름을 검증합니다.
- 실제 서버 연결은 `src/repositories/apiRepositories.ts`의 adapter와 `docs/CLOUD_SCHEMA.md` 스키마를 기준으로 붙이면 됩니다.

## 데이터 백업

- 프로젝트는 제출 패키지의 `.webband.json`으로 백업할 수 있습니다.
- 커스텀 레슨은 레슨 빌더에서 JSON으로 내보내고 다시 가져올 수 있습니다.
- 교사용 제출 현황은 CSV로 내보낼 수 있습니다.
