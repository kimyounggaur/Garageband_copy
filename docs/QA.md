# QA 체크리스트

## 자동 확인

```bash
npm run build
npm run qa:smoke
```

`qa:smoke`는 migration, mission 평가, review 요약, 음악 이론 분석, 오디오 편집 설정, 수업 repository, GitHub Pages 빌드 경로가 코드에 존재하는지 확인합니다.

## 브라우저 수동 시나리오

1. 새 프로젝트를 만들고 이름을 바꾼 뒤 저장합니다.
2. 루프를 타임라인에 추가하고 재생합니다.
3. 미디 클립을 만들고 피아노롤에서 노트를 추가, 이동, 삭제합니다.
4. 마이크 녹음 또는 오디오 업로드로 샘플을 만들고 타임라인에 재삽입합니다.
5. 오디오 클립의 앞부분/뒷부분 자르기, 음량 보정, 페이드, 나누기를 적용합니다.
6. 교사 보기에서 반과 학생을 만들고 과제를 배정합니다.
7. 학생 보기에서 학생을 선택하고 과제를 시작합니다.
8. 레슨 미션을 완료하고 검토 모드에서 점수와 다음 행동 추천을 확인합니다.
9. 제출 패키지를 만들고 교사 보기에서 제출 상세와 피드백 저장을 확인합니다.
10. 레슨 빌더로 커스텀 레슨을 만들고 내보내기/가져오기를 확인합니다.
11. 로컬 저장소와 모의 클라우드 저장소를 전환해도 앱이 멈추지 않는지 확인합니다.
12. `docs/index.html` 또는 GitHub Pages 배포 URL에서 앱이 정상 로드되는지 확인합니다.

## 회귀 위험

- IndexedDB 스키마가 올라가므로 기존 프로젝트가 `normalizeProject`를 거쳐 열리는지 확인합니다.
- 오디오 Blob은 브라우저 저장소 용량에 영향을 받습니다.
- 커스텀 레슨은 현재 브라우저 저장소 기반이므로 다른 기기와 공유하려면 JSON export/import가 필요합니다.
